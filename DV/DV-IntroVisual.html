9  Visualizing Data

This section provides basic ideas on data visualization.

9.1 What is Data Visualization?

Data visualization (or data viz) is a non-text-based representation of data that enhances our understanding of it. The goal is to take numbers, which by themselves can be overwhelming, and turn them into something visual so the human brain can quickly identify patterns, trends, and outliers. Thus, data visualization is a core component of data analysis, and there are many packages and approaches available when working in Python.

Data visualization is not just numbers, and it’s not just pictures — it’s the fusion of the two. A spreadsheet of values isn’t a visualization. A pretty poster with no data isn’t a visualization. But a well-designed chart, map, or graph that makes the numbers clearer? That’s data visualization. Another way to think about data visualization is as a visual language. The sender encodes information into a chart, and the receiver decodes it to find meaning. If the visualization works, it speeds up comprehension and reveals insights that would otherwise stay hidden in the numbers.

A classic example of why visualization matters is Anscombe’s Quartet. This is a set of four datasets that, on paper, look nearly identical — they all have the same mean, the same standard deviation, and the same correlation. If you just looked at the numbers, you might assume the datasets are essentially the same. But when you plot them out, you see they’re completely different — one is a linear relationship, another is curved, one has an outlier, and another forms almost a perfect line with a single point skewing the data.

Show the code 
# packages
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt


# Define the four datasets
x = [10, 8, 13, 9, 11, 14, 6, 4, 12, 7, 5]
y1 = [8.04, 6.95, 7.58, 8.81, 8.33, 9.96, 7.24, 4.26, 10.84, 4.82, 5.68]
y2 = [9.14, 8.14, 8.74, 8.77, 9.26, 8.10, 6.13, 3.10, 9.13, 7.26, 4.74]
y3 = [7.46, 6.77, 12.74, 7.11, 7.81, 8.84, 6.08, 5.39, 8.15, 6.42, 5.73]
x4 = [8, 8, 8, 8, 8, 8, 8, 19, 8, 8, 8]
y4 = [6.58, 5.76, 7.71, 8.84, 8.47, 7.04, 5.25, 12.50, 5.56, 7.91, 6.89]

datasets = {"I": (x, y1), "II": (x, y2), "III": (x, y3), "IV": (x4, y4)}

# Create subplots
fig, axs = plt.subplots(
    1, 4, sharex=True, sharey=True, figsize=(12, 3),
    gridspec_kw={"wspace": 0.08, "hspace": 0.08},
)

axs[0].set(xlim=(0, 20), ylim=(2, 14))
axs[0].set(xticks=(0, 10, 20), yticks=(4, 8, 12))

# Plot each dataset with regression line and summary stats
for ax, (label, (x, y)) in zip(axs.flat, datasets.items()):
    ax.text(0.1, 0.9, label, fontsize=20, transform=ax.transAxes, va="top")
    ax.tick_params(direction="in", top=True, right=True)
    ax.plot(x, y, "o")

    # Linear regression
    slope, intercept = np.polyfit(x, y, deg=1)
    ax.axline(xy1=(0, intercept), slope=slope, color="r", lw=2)

    # Statistics box
    stats = (
        f"$\\mu$ = {np.mean(y):.2f}\n"
        f"$\\sigma$ = {np.std(y):.2f}\n"
        f"$r$ = {np.corrcoef(x, y)[0][1]:.2f}"
    )
    bbox = dict(boxstyle="round", fc="blanchedalmond", ec="orange", alpha=0.5)
    ax.text(
        0.95, 0.07, stats, fontsize=9, bbox=bbox,
        transform=ax.transAxes, horizontalalignment="right",
    )

plt.suptitle("Anscombe's Quartet")
plt.show()

This example shows why visualization is so important. Without it, we might make the mistake of treating very different datasets as though they were identical. With visualization, we see the real story immediately. So, to sum it up: data visualization is about making numbers visual so we can learn something from them that would otherwise be invisible.

9.1.1 Data Visualization vs. Infographic

A data visualization requires actual data, and it represents that data visually in a way that makes it easier to understand. It’s usually more about graphs, charts, and geometric elements that highlight relationships among numbers.

An infographic, on the other hand, is more like a designed collection of facts. It often uses icons, images, and a lot of text. It may include a little data, but it’s not always data-driven. Typically, an infographic is divided into sections, and each section is with text and pictures. Some section may or may not include a data visualization. Many of sections may just list out facts where the visuals are to catch your eye, not necessarily to represent data.

Nicely designed posters with a few numbers on them aren’t really data viz. — Amanda Cox, an editor at The New York Times

Infographics are great for storytelling and communication. But if you’re trying to reveal insights hidden in numbers, then data visualization is what you need.

9.2 Why Use Data Visualizations?

There are four key reasons to use data visualization, instead of just sticking with numbers in tables, or long paragraphs of text:

  • Speed: Data visualizations allow us to grasp meaning within just a few seconds. Think about our attention spans today — they’re shorter than ever where we need to deal with lots of data every day in the fast-moving world. A well-designed chart communicates in a glance what would take several sentences or even paragraphs to explain in words.

  • Comprehension: Complex information becomes easier to understand when it’s visual. Visuals help us uncover trends, patterns, and outliers that would be hard to notice in a spreadsheet. For example, if I show you a time-series line chart, you’ll immediately see the rise and fall of values over time — something that’s much harder to perceive in raw numbers.

  • Appeal: Visuals are attention-grabbing. Text and tables can feel dry, but a good chart or map is more engaging. It can make your message more compelling and memorable.

  • Retention: Our brains are wired to remember images. If you present data visually, people are far more likely to recall it later compared to when they just read the numbers.

Visuals speed up understanding, clarify complexity, capture attention, and help us remember the message. That’s why visualization is such a powerful communication tool.

9.3 What Makes a Good Data Visualization?

There are five main qualities that can lead to good data visualizations:

  • Accuracy: A good visualization must be truthful. It should not misrepresent data or exaggerate differences. For example, manipulating the scale of an axis can easily mislead — that’s something we must avoid.

  • Clarity: A visualization should be easy to understand. If it takes several minutes for someone to figure out what your chart is saying, then it’s not doing its job.

  • Relevance to the audience: Always consider the people who will see your visualization. Ask yourself: ‘So what?’ Why should they care? A good visualization answers that question clearly.

  • Simplicity: Show only what’s necessary. Remove clutter. Every line, label, or shape should serve a purpose. If something doesn’t help people understand the data, it doesn’t belong.

  • Storytelling: A strong visualization tells a story. It’s not just about presenting numbers — it’s about guiding the audience to a conclusion, an insight, or a key message.

“Beauty is desirable, but I’m not excited by pretty charts that are also pretty useless.” — Graham Wills, a data science expert at IBM

The goal isn’t just to make something pretty — it’s to make something meaningful.

9.4 Planning Your Data Visualization

9.4.1 The Story

When planning a data visualization, the very first step is to identify the story in your data.

Let’s consider an example. Here, we see data from Demographic and Health Surveys showing the percentage of demand for family planning satisfied with modern methods across several years. Notice what happens: in 1993, the rate was just about 40 percent. For nearly a decade, it barely moved — stagnating below 50 percent. But in recent years, something changed. The rate climbed sharply, surpassing 50 percent and eventually reaching over 70 percent.

Now, if I had simply shown you this as a table of numbers, that story might not have been obvious. But as a visual, the trend jumps out. That’s the power of visualization — it not only communicates information, it actually helps us discover the story hidden in the numbers.

So, when you plan your visualization, ask yourself: what is the key message or story here? Once you know that, the design decisions become much easier.

NoteClarify the Message

A graphic may tell a thousand words, but you must decide which words.

  • Ask yourself: “What does this plot tell the viewer?”
  • Identify the key takeaways that need to be immediately apparent in the graphic
  • Ensure the viewer is left with no ambiguity about your intended message

9.4.2 The Audience

The second step in planning is to think carefully about your audience. Who are you creating this visualization for?

You can start by considering a few factors:

  • Where are they from? What do they already know about the subject?
  • Are they experts, policymakers, or members of the general public?
  • Do they have access to high-speed internet, or will they only see a printed version?
  • Are they likely to be skeptical, or are they already invested in the topic?

By finding some answers to these questions, you can define the expected audience.

After that, most importantly, you should ask yourself:

  • Why should they care about this visualization?

If the answer isn’t clear, then the visualization isn’t doing its job.

Here’s an example: imagine you’re presenting data on health insurance coverage in US. If your audience is local policymakers for early career development, you may want to highlight patterns or distribution in specific age groups, and keep the design straightforward. But if your audience is a federal agency, you might emphasize comparisons across various characteristics.

In short: the story may stay the same, but how you frame it — what you emphasize, what you simplify — depends entirely on who’s in your audience.

9.4.3 Accuracy

The third key planning step is accuracy. Accuracy in data visualization is non-negotiable. Misrepresenting data — even unintentionally — can destroy trust.

Here are some common mistakes to watch out for:

  • Percent change vs. point change. These are not the same, and mixing them up can dramatically distort the story.
  • Bubble charts sized by diameter, not area. If you scale bubbles incorrectly, the differences look much larger than they actually are.
  • Inconsistent units. Mixing currencies, time periods, or measurement systems without clarification confuses the reader and undermines credibility.
  • Oversimplification. Stripping out too much context can make data misleading or meaningless.
  • Truncated axes. If you cut off the y-axis and start at a higher number, small differences can appear exaggerated. For example, showing a bar chart that starts at 30 makes a subtle difference look enormous.

The bottom line is this: your audience should take the message based on the truth of the data, not a distorted impression. Accuracy is what gives your visualization integrity.

9.5 Design Principles

It’s good to know some tips and principles in designing data visualization.

9.5.1 Ink

One of the most important design principles in visualization is what Edward Tufte called the data-ink ratio. Think of it this way: every bit of ink, or every pixel, on your chart should be doing work to help communicate the data. If it’s not helping, it’s distracting.

So how do we improve the data-ink ratio? Start by stripping away unnecessary elements. For example, the bar chart is well decorated with lots of colors and backgrounds. But, the same information can be delivered more compactly with higher data-ink ratio.

And, even further, we can make this chart slimer by removing one more from this chart. The gridlines don’t actually help us estimate values precisely, and they may not add value to this visualization. Instead, we can label the bars directly and drop the y-axis altogether. That alone makes the graphic cleaner.

However, this kind of removal may not work well for other visualizations in such cases that a larger number of items are included in the chart so that it becomes harder to compare each other by taking the numbers from labels for the bars.

We can also lighten labels, simplify color schemes, and reduce decorative elements — things like shading, 3D effects, or drop shadows. These don’t clarify the message; they clutter it. The goal isn’t to make the chart boring — it’s to make sure every line, every word, and every shape contributes to understanding the data. And if it doesn’t, delete it. That’s how you use your ink strategically.

9.5.2 Texts

Even though we want visuals to do most of the work, good text is still essential in data visualization.

Here are some tips:

  • Use text and labels to clarify, not clutter.
    • Include labels if they help people read the chart quickly.
      • Axis labels – Always include them, and if there are units, state them clearly (e.g., “Salary (2015 USD)”).
      • Tick labels – Tailor them to the scale (linear vs. logarithmic).
      • Titles – Use them if axes and the plot alone do not fully convey the message.
    • Use text annotations to point out key features.
  • Make your titles work harder.
    • Titles should communicate the key takeaway. So that, the audience can immediately know the message you want them to walk away with.
  • Choose fonts for readability. Sans serif fonts — like Helvetica, Arial, or Gill Sans — are clean, modern, and easier to read on screens.
    • Use emphasis sparingly. All caps, bold, italics, or larger sizes can draw attention, but don’t overuse them. If everything is emphasized, nothing stands out.
    • Stay consistent. Switching fonts or styles across slides or graphics makes the design look messy.

9.5.3 Colors

Color is another powerful tool in your design toolbox — but only if you use it thoughtfully.

Our brains process color much faster than shapes or text, so color can guide attention instantly.

To make it work, the following principles would help us.

  • First, use darker or brighter colors to mean ‘more.’
    • Stronger colors should correspond to higher numbers or greater importance.
    • Use gray for neutral or background data.
      • By doing so, it will help audience keep the focus where you want to emphasize.
      • Fading less important elements with transparency is a similar option.
  • Second, every color should have a purpose. Don’t add colors just because they look nice. Each one should signal a category, a comparison, or a level of emphasis.
    • Also, keep palettes consistent. If blue means ‘Region A’ on one chart, it should mean ‘Region A’ everywhere else.
    • Besides, contrast matters. Higher contrast makes objects stand out more. If two bars are meant to be compared directly, don’t use two nearly identical shades.
  • Finally, be mindful of symbolism. Colors may carry meaning beyond the chart. Red often signals danger, urgency, or loss. Blue conveys calm, trust, or stability. Yellow can mean optimism — or caution. Think about what your colors might communicate emotionally as well as numerically.

9.6 Suggested Workflow & Resources

When it comes to actually building a data visualization, it’s helpful to have a process to guide you.

Here’s a simple workflow you can follow:

  1. Experiment.
    • Start by trying different types of visuals.
    • Don’t lock yourself into one chart too early.
    • Sometimes, the story in your data only becomes clear after you’ve tried out a few formats.
  2. Decide.
    • Once you’ve explored, decide which type of visual best fits your data and the story you want to tell.
      • Is it a line chart to show trends over time?
      • A bar chart for comparisons?
      • A map for geographic differences?
  3. Refine.
    • Next, experiment with design options.
    • This is where trial and error comes in — adjust colors, labels, scales, and layouts until the message is clear and uncluttered.
  4. Build the final version.
    • After testing and refining, put everything together into a polished visualization that communicates your message effectively.

Key takeaway: building a strong visualization is not a one-shot effort — it’s an iterative process of experimenting, refining, and polishing.

9.7.2 Python Data Visualization Packages

Python has a rich ecosystem of visualization packages, which can be categorized in two broad approaches to creating visualizations with code:

  1. Imperative – You specify step by step what to draw.
    • Pros: Maximum flexibility and control.
    • Cons: Often verbose; more effort for common plots.
  2. Declarative – You declare what you want, and the library handles the details.
    • Pros: Quick and concise for standard chart types.
    • Cons: Requires data in the correct format; customization can be limited.

Depending on your goals and data, you can choose between them and accordingly a python package. The following sections introduce some popular python packages. For a comprehensive list, see the PyViz Tools Overview.

9.7.2.1 Core Visualization Packages

Here are the core libraries you are most likely to use either directly or indirectly (as many other high-level libraries are built on these core libraries):

Matplotlib is the most important and widely used visualization library in Python.

  • Key Features:
    • Imperative approach – build plots piece by piece for complete control.
    • Highly flexible but can be verbose for complex plots.
    • Supports static plots, diagrams, animations, and 3D visualizations (3D should be used sparingly for clarity).
  • When to Use:
    • You need fine-grained control over every visual element.
    • You are making bespoke or unusual charts.
    • You require careful graphical design or incremental plot building.

Plotly is a declarative-oriented library designed for interactive visualizations.

  • Key Features:
    • Generates interactive charts with tooltips and zooming.
    • Ideal for dashboards and web applications.
    • Can export static images but shines in browser-based interactivity.
  • When to Use:
    • You are building a data dashboard or web-based visualization.
    • You need interactive plots to explore data beyond static figures.

Bokeh is another interactive visualization library for Python.

  • Key Features:
    • Enables browser-based, interactive plotting.
    • Can handle streaming and real-time data.
    • Integrates with Jupyter notebooks, dashboards, and web apps.
  • When to Use:
    • You need interactive visualizations with real-time or large data handling.
    • You want standalone HTML outputs for sharing results without extra dependencies.

9.7.2.2 Other Libraries

Python has many specialized visualization packages that complement the core ones. These are useful for statistical plotting, interactive dashboards, machine learning visualization, or domain-specific applications.

  • Pandas Built-in plotting provides quick, convenient plots:
    • Syntax: df.plot.* (e.g., df.plot.scatter).
    • Internally uses Matplotlib.
    • Excellent for fast exploratory data analysis (EDA), but not suitable for polished publication figures.
  • Seaborn is a high-level statistical visualization library built on Matplotlib:
    • Works best with tidy data (one row per observation, one column per variable).
    • Provides high-level functions for common statistical plots (e.g., box plots, violin plots, heatmaps).
    • Ideal for exploratory analysis and quick statistical graphics.
  • Altair is a declarative, web-oriented library:
    • Produces beautiful, minimalist charts with minimal code.
    • Best for interactive, browser-friendly visualizations.
    • Built on Vega-Lite widely used in newsroom data visualization.

Beyond the core and high-level packages, Python offers many specialized tools for niche visualization needs:

  • proplot: Lightweight Matplotlib wrapper for publication-quality plots.
  • seaborn-image: Brings Seaborn-like workflows to image data.
  • Lit: Visualization and interpretability for NLP models.
  • Wordcloud: Generates word clouds (use sparingly!).
  • VisPy: GPU-accelerated visualization for very large datasets.
  • HoloViews: Simplifies data-to-visualization workflows; builds on Bokeh and Matplotlib.
  • chartify: Quick, high-level plotting library from Spotify.
  • palettable: Additional color palettes for Matplotlib and Seaborn.
  • colorcet: Perceptually uniform color maps** for accurate data perception.
  • missingno: Visualizes patterns of missing data in your dataset.
