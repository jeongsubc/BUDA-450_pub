15  Model Validation

Model validation estimates how well a trained model will perform on unseen data.

15.1 Data Preparation

Weâ€™ll use the Breast Cancer Wisconsin dataset from sklearn.datasets. It contains 569 samples with 30 numeric features, labeled as benign or malignant.

import numpy as np
import pandas as pd
from sklearn.datasets import load_breast_cancer

# Load dataset

X_full = bc.data

#print(X_full.head())

# Basic dimensions

p = X_full.shape[1]  # number of input attributes
print(f"n = {n}, p = {p}")
n = 569, p = 30

15.2 Cross-validation

Cross-validation (also called out-of-sample testing) assesses generalization by training on one subset of the data and evaluating on a disjoint subset. This separation prevents information leakage and gives a trustworthy estimate of how the model will perform on new data.

In practice, cross-validation is commonly performed:

  • Choose a validation scheme (Holdout, Monte Carlo CV, or k-Fold CV).
  • Tune hyperparameters (here, the k in k-NN) using only the training/validation data.
  • Lock in the chosen hyperparameter(s) and evaluate once on the test set.

This section introduces three widely used strategies in Python:

  • Holdout (single split):
    • Partition once into train/validation/test.
    • Fast and simple; higher variance because results depend on a single split.
  • Monte Carlo Cross-Validation (repeated random subsampling / LGOCV):
    • Repeat random train/validation splits many times and average metrics.
    • Reduces variance by averaging across multiple random splits.
  • k-Fold Cross-Validation: Split data into k folds and rotate the validation fold.
    • Divide data into k folds; train on k-1, validate on the remaining fold, and rotate.
    • Every sample serves as validation exactly once; efficient for small datasets.

In the following examples, weâ€™ll use these procedures/strategies to tune the number of neighbors (k) for a k-Nearest Neighbors (k-NN) classifier and then report performance on a held-out test set.

15.3 Holdout Method

15.3.1 Data Partitioning

Letâ€™s partition the dataset into three disjoint sets: 60% for training, 20% for validation, and 20% for testing. The validation set is used for model/parameter selection; the test set is held back for a final, unbiased estimate.

Many operations use randomness, so results change across runs unless you fix the seed.

NumPy seed (manual RNG control). A seed sets the starting point of a pseudo-random sequence. Re-creating a generator with the same seed reproduces the same sequence (and therefore the same split).

# Without fixing a seed: results vary between runs
print("No seed â€” 1st run:", np.random.permutation(10))
print("No seed â€” 2nd run:", np.random.permutation(10))
No seed â€” 1st run: [1 3 7 5 0 8 4 6 9 2]
No seed â€” 2nd run: [5 1 7 3 2 0 9 6 4 8]
# With a fixed seed using the Generator API: results are repeatable
rng1 = np.random.default_rng(123)
print("Seeded â€” 1st run:", rng1.permutation(10))

rng2 = np.random.default_rng(123)
print("Seeded â€” 2nd run:", rng2.permutation(10))  # identical to the 1st run
Seeded â€” 1st run: [7 4 0 2 1 5 6 8 3 9]
Seeded â€” 2nd run: [7 4 0 2 1 5 6 8 3 9]

scikit-learn random_state. Many scikit-learn functions have a random_state parameter that controls their internal shuffling/sampling. Pass a fixed integer to make those operations reproducible. Setting a NumPy seed alone does not guarantee deterministic scikit-learn behavior unless you also set random_state when available.

from sklearn.model_selection import train_test_split

# Different each run (no random_state)
X_a, X_b, y_a, y_b = train_test_split(X_full, y_full, test_size=0.20)

# Identical across runs (fixed random_state)
X_tr_va, X_te, y_tr_va, y_te = train_test_split(
    X_full, y_full, test_size=0.20, random_state=123
)

Rule of thumb:

  • Use NumPyâ€™s seed (via default_rng(seed)) to control your own random draws.
  • Set random_state on scikit-learn functions to control their randomness.

15.4 Monte-Carlo Cross-Validation

Monte-Carlo (repeated random subsampling) CV estimates generalization by repeatedly splitting the data at random, training on a train+validation subset, and evaluating on a held-out test subset within the pool. Scores are then averaged across repeats.

15.4.1 Data Partitioning

The snippet below shows one random split to produce the train+validation pool and a test set (kept untouched for final evaluation):

  • fix a random seed for reproducibility,
  • create a random permutation of indices,
  • take the first n_tr + n_va rows as train+validation,
  • use the remaining rows as the test set.
# Reproducible random permutation of 0..n-1
rng = np.random.default_rng(2024)
idx_rand = rng.permutation(n)

# Indices for train+validation and test
idx_tr_va = idx_rand[:(n_tr + n_va)]   # training + validation
idx_te    = idx_rand[(n_tr + n_va):]   # test

# Subset the data
X_tr_va = X_full.iloc[idx_tr_va]
y_tr_va = y_full.iloc[idx_tr_va]

X_te = X_full.iloc[idx_te]
y_te = y_full.iloc[idx_te]

# Quick check

For the Monte-Carlo cross-validation1, letâ€™s use scikit-learnâ€™s ShuffleSplit() that defines how to randomly split data for cross-validation inside the train+validation pool:

from sklearn.model_selection import ShuffleSplit

# Same ratios/iterations as before
ratio_tr_va = 0.75      # proportion used for training in each split
no_iterations = 10      # number of random splits

# Leave-Group-Out style CV without stratification
cv = ShuffleSplit(
    n_splits=no_iterations,
    test_size=1 - ratio_tr_va,
    random_state=2024
)

15.4.2 Model Selection with Cross-validation Result

Then, you can Use GridSearchCV() to evaluate multiple hyperparameters across repeated random splits (our Monte-Carlo / LGOCV setup via cv = ShuffleSplit(...)) and automatically select the best model.

Here we search over (k {2,5,10}) for k-NN, using accuracy as the validation metric.

from sklearn.model_selection import GridSearchCV

# Model
knn = KNeighborsClassifier()

# Hyperparameter grid (keys match the estimator's __init__ arguments)
param_grid = {"n_neighbors": [2, 5, 10]}

# Grid search over k using the Monte-Carlo splits defined in `cv`
grid = GridSearchCV(
    estimator=knn,
    param_grid=param_grid,
    scoring="accuracy",
    cv=cv,                       # ShuffleSplit from the previous block
    return_train_score=False
)

# Fit on the train+validation pool
grid.fit(X_tr_va, y_tr_va)

15.5 k-fold Cross-Validation

In k-fold CV, the data are split into k equal folds. For each round, one fold acts as the validation set and the remaining kâˆ’1 folds form the training set. Metrics are averaged across the k rounds. Compared with a single holdout, k-fold CV uses all samples for validation exactly once, often yielding a more stable estimate.

For k-fold cross validation, the similar procedure and codes can be used.

Below we keep the rest of the workflow identical to the previous sections, and only change the CV design to KFold() with k.

# k-fold cross-validation (k=5)
from sklearn.model_selection import KFold

kfold = 5
cv = KFold(n_splits=kfold, shuffle=True, random_state=2024)

param_grid = {"n_neighbors": [2, 5, 10]}
knn = KNeighborsClassifier()

grid = GridSearchCV(
    estimator=knn,
    param_grid=param_grid,
    scoring="accuracy",
    cv=cv,
    n_jobs=-1,
    return_train_score=False
)

# Fit on the train+validation pool (X_tr_va, y_tr_va from earlier)
grid.fit(X_tr_va, y_tr_va)

# Summary of cross-validated performance per k
cv_summary = (
    pd.DataFrame(grid.cv_results_)[
        ["param_n_neighbors", "mean_test_score", "std_test_score", "rank_test_score"]
    ]
    .sort_values("rank_test_score")
    .rename(columns={
        "param_n_neighbors": "k",
        "mean_test_score": "mean_acc",
        "std_test_score": "std_acc",
        "rank_test_score": "rank"
    })
)
print("\nCross-validated accuracy (k-fold):\n", cv_summary)
