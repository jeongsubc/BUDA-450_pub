1.2 Preliminaries

1.2.1 Why Python?

Python is one of the most popular programming languages.

It‚Äôs one of the easiest to learn. Further. Also, it can be used for a wide range of tasks as a general-purpose language. This combination is why people often say Python has a ‚Äúlow floor and high ceiling.‚Äù

Further, it‚Äôs also very versatile as it can cover various tasks (e.g., basic calculations to breakthrough innovations like large language models) in many domains, including machine learning, data science, web development, automation, engineering, etc., and accordingly across the industry, academia, and the public sector.

Its simplicity and readability make it a common first language for students, while its deep ecosystem of libraries makes it suitable for advanced tasks.

For business analytics, learning Python is especially valuable. Once you become familiar with Python (or any language), learning other (more specialized) languages like R or C++ becomes much easier. Most of the programming concepts, which you‚Äôll see in this course such as variables, loops, and data structures, similarly apply across most languages.

Besides, Python also shines in cloud computing environments, thanks to strong support from cloud providers and its rich ecosystem of open-source packages.

In summary, Python is an outstanding language for analytics, data science, and various applications in business. It‚Äôs no surprise that it consistently ranks among the most popular programming languages in the world!

1.2.2 Concepts

Before diving into coding, it‚Äôs important to understand a computational environment‚Äîthe setup that allows you to write and run Python code. In this section, let‚Äôs briefly understand the key elements of a computational environment for coding.

A computational environment typically includes elements that together define where and how your code will run:

  • The operating system you‚Äôre working on (e.g., macOS Catalina)
  • The version of the programming language (e.g., Python 3.10)
  • Installed packages or libraries (e.g., pandas 2.1.0)

To set up a working Python environment, you‚Äôll need:

  • A computer with an operating system (e.g., Windows, macOS, Linux, or a cloud-based platform)
  • An installation of Python, the programming language that enables the computer to interpret and execute the code
  • Various packages that extend Python‚Äôs capabilities for various tasks
  • And, an Integrated Development Environment (IDE) that is a tool to write and execute code (in a convient way)

Let‚Äôs walk through each of these in more detail.

1.2.2.1 Operating System

Almost all the code you‚Äôll see can be run on all three of the major operating systems: Windows, MacOS, and Linux, so it doesn‚Äôt matter much what operating system you‚Äôre using. Almost all cloud services use Linux.

If you haven‚Äôt yet decided which operating system to use, this book recommends either Linux or MacOS because, in a very small number of cases, you‚Äôll find it easier to run the most advanced code on them rather than on Windows. Don‚Äôt panic if you have Windows already though‚Äîmost things will work just fine and it tends to be power users who run into this problem. (If you‚Äôre not familiar with Linux, it‚Äôs a free operating system that is also widely used for cloud services and, while it used to have a reputation as being fearsomely difficult for beginners, some modern Linux distributions, such as Ubuntu, are pretty user-friendly.)

If you have Windows and you want to use Linux or Mac but don‚Äôt want to shell out for a new computer, there are a couple of options. One is to use the Windows Subsystem for Linux. It‚Äôs essentially a Linux operating system that installs alongside and integrates with your existing Windows operating system. This allows you to run code as if you were using Linux. You can get WSL for free from Microsoft. Another option (easier but can be more expensive) is to use an online cloud service, an option we‚Äôll return to shortly.

1.2.2.2 Python interpreter

Python is both a programming language that humans can read, and a language that computers can read, interpret, and then carry out instructions based on. For a computer to be able to read and execute Python code, it needs to have a Python interpreter installed. There are lots of ways to install a Python ‚Äúinterpreter‚Äù on your own computer, this book recommends the uv distribution of Python for its flexibility, simplicity, and features. Cloud services often come with a Python interpreter installed, and we‚Äôll see shortly how to install one on your own computer.

1.2.2.3 Integrated Development Environment (IDE)

An integrated development environment (IDE) is a software application that provides a few tools to make coding easier. The most important of these is a way to write the code itself! IDEs are not the only way to programme, but they are perhaps the most useful.

If you have used Stata or Matlab, you may not have realised it, but these analytical tools bundle the interpreter and the IDE together. But they are separate things: the interpreter is a way of processing your instructions, the IDE is where you write those instructions.

There are a lot of integrated development environments (IDEs) out there, including PyCham, IDLE, Visual and Studio Code. In this course, Jupiter will be used as it works on all major operating systems and is one of the most popular.

1.2.2.4 Packages

A Python package is a collection of functions, data, and documentation that extends the capabilities of an installed version of Python. Using packages is key to most data science because most of the functionality we‚Äôll need comes from extra packages. You‚Äôll see statements like import numpy as np at the start of many Python code scripts‚Äîthese are instructions to use an installed package (here one called numpy) and to give it a shortened name (np, for convenience) in the rest of the script. The functions in the numpy package are then accessed through syntax like np.; for example, you can take logs with np.log(x) where x is a variable containing a number. You need only install packages once, but you must import them into each script you need to use them in. We‚Äôll see more on how to both install and use packages in subsequent chapters.

1.2.2.5 Typical workflow

The typical workflow for analysis with code might be something like this:

  • Open up your integrated development environment (IDE)
  • Write some code in a script (a text file with code in) in your IDE
  • If necessary for the analysis that you‚Äôre doing, install any extra packages
  • Use the IDE to send bits of code from the script, or the entire script, to be executed by Python and add-on packages, and to display results
  • (once the project is complete) ensure the script can be run from top to bottom to re-produce your analysis

We‚Äôll see two ways to achieve this workflow:

  1. Installing an IDE, a Python interpreter, and any extra Python packages on your own computer
  2. Using a computer in the cloud that you access through your internet browser. Cloud computers often have an IDE and Python built-in, and you can easily install extra packages in them too. However, you should be aware that the cloud service we recommend has a 60 hours / month free tier. Beyond this you‚Äôll need to pay for extra hours.

You should pick whichever you‚Äôre more comfortable with! Eventually, you‚Äôll probably try both.

1.3 Setting Your Own Computer

The following instruction is for a python environment on your own computer. The overall installation is as follow:

  1. (Optional) Install a software for virtual environments (Anaconda), and create a virtual environment
  2. Install Python (in the virtual environment)
  3. Install an IDE (Jupiter Notebook) (in the virtual environment)

1.3.1 Virtual Environment

Different projects often require different environment setups. That‚Äôs when a virtual environment comes in handy, which helps to work on isolated Python environments where you can freely create/delete them (with no risk of messing up your entire computer).

Also, considering rapid version updates, an easy way to manage a python environment for the compatibility of python and many packages is building it on a virtual environment.

Why Use Conda to Install Python?

Installing Python through Conda (via Anaconda or Miniconda) is highly recommended for this course, especially for users working with data science and analytics tools.

Here‚Äôs why:

  • Isolated Environments: Conda allows you to create separate environments for different projects. Each environment can have its own version of Python and packages‚Äîpreventing version conflicts.
  • Better Package Management: Unlike pip (Python‚Äôs default package manager), Conda can install not just Python packages but also system-level dependencies like C or Fortran libraries. This is especially helpful for scientific packages like numpy, scipy, and pytorch.
  • Cross-Platform Compatibility: Conda works consistently across Windows, macOS, and Linux. It is also used by most cloud data science environments, helping ensure reproducibility.
  • Fast Setup with Anaconda: The Anaconda distribution includes over 250 pre-installed data science packages (e.g., pandas, matplotlib, scikit-learn, jupyterlab), making it ideal for beginners and fast onboarding.
  • Scientific Computing Support: Many data science and machine learning tools depend on optimized compiled libraries. Conda handles these dependencies more reliably than pip.
  • Reproducible, portable environments that ‚Äújust work,‚Äù especially in data science: Conda is the most robust choice.

1.3.3.1 Installing Python

To download and install Python, we‚Äôll use the uv ‚Äúdistribution‚Äù of Python, which is available on all major operating systems. To install it, follow the instructions at this website. Unlike installing normal programmes, we‚Äôre going to use the command line to install Python. Linux, Mac, and Windows all have built-in command lines: search for Terminal on Mac or Linux, and Powershell on Windows. These apps will bring up boxes that you can type commands in. As of the time of writing, the commands are:

for Linux and Mac, and

for Windows. Hit return to execute the commands.

Once you have installed uv, you can check it‚Äôs installed properly by running uv --version. You should see a message pop up that says ‚Äúuv‚Äù and then the latest version number.

We didn‚Äôt actually download Python yet‚Äìwe downloaded uv, which is going to manage our Python environment. So next we need to run

uv python install

1.3.6.2 Running Python code

Now you will create and run your first code. If you get stuck, there‚Äôs a more in-depth tutorial over at the VS Code documentation.

In Visual Studio Code, click on the ‚ÄúExplorer‚Äù symbol (some files on the left-hand side of the screen) to bring up a file explorer. Check you‚Äôre in a good location on your computer to try things out and, if not, change the folder you‚Äôre in using File -> Open Folder until you‚Äôre happy.

Now open up a terminal within Visual Studio Code. Make sure it‚Äôs the same type of terminal that you installed uv in, eg use Powershell on Windows (you can select to use different types of terminals.) The shortcut to open the terminal panel is Ctrl + ` on Linux and Windows, and Cmd + ` on Mac. The terminal panel appears along the bottom of VS Code. Run uv init in the terminal to initiate an installation of Python. This Python installation only lives in your current folder. Then, to install the package we need for this section run uv add jupyter. (We will cover the terminal and installing packages in much more detail shortly.)

Now, still with the explorer panel open on the left hand side, click on the symbol that looks like a blank piece of paper with a ‚Äú+‚Äù sign on it. This will create a new file, and your cursor should move to name it. Name it hello_world.py. The file extension, .py, is very important as it implicitly tells Visual Studio Code that this is a Python script.

In the Visual Studio Code editor, add a single line to the file:

print('Hello World!')

Save the file.

If you named this file with the extension .py then VS Code will recognise that it is Python code and you should see the name and version of Python pop up in the bar at the bottom of your VS Code window. (You can have multiple versions of Python installed‚Äîif you ever want to change which Python version your code uses, click on the version shown in the bar and select the version you want.)

Alright, shall we actually run some code? Select/highlight the print("Hello world!") text you typed in the file and right-click. You‚Äôll get a lot of options here, but the one you want is ‚ÄúRun Selection/Line in Interactive Window‚Äù.

This should cause a new ‚Äòinteractive‚Äô panel to appear within Visual Studio Code, and, hey presto you should see:

Hello world!

The interactive window is a convenient and flexible way to run code that you have open in a script or that you type directly into the interactive window code box. The interactive window will ‚Äòremember‚Äô any variables that have been assigned (for examples, code statements like x = 5), whether they came from running some lines in your script or from you typing them in directly. Working with the interactive window will feel familiar to anyone who has used Stata, Matlab, or R. It doesn‚Äôt require you to write the whole script, start to finish, ahead of time. Instead, you can jam, changing code as you go, (re-)running it line by line.

It would be cumbersome to have to right-click every time we wanted to run some code, so we‚Äôre going to make a keyboard shortcut to send whatever code is highlighted to the interactive window to be executed. To do this:

  • Open up the Visual Studio Code configuration menu (the cog on the lower left-hand side)
  • Go to Settings
  • Type ‚Äújupyter send‚Äù in the box to make an entry ‚ÄúInteractive Window > Text Editor: Execute Selection‚Äù appear
  • Ensure the box next to this entry is ticked

Now return to your script, put your cursor on the line with print("Hello world!") on, and hit Shift+Enter. You should see ‚ÄúHello world!‚Äù appear again, only this time, it was much easier.

1.3.7 Packages and how to install them

Packages (also called libraries) are key to extending the functionality of Python and it won‚Äôt be long before you‚Äôll need to install some extra ones! There are packages for geoscience, for building websites, for analysing genetic data, and, yes, of course, for economics. Packages are typically not written by the core maintainers of the Python language but by enthusiasts, firms, researchers, academics, all sorts! Because anyone can write packages, they vary widely in their quality and usefulness. There are some that are key for an economics workflow, though, and you‚Äôll be seeing them again and again.

The three Python packages numpy, pandas, and maplotlib, which respectively cover provide numerical, data, and plotting functionality, are ubiquitous. So many scripts begin by importing them, as in the tweet above!

Python packages don‚Äôt come built-in (by definition) so you need to install them (just once, like installing any other application), and then import them into your scripts (whenever you use them in a script). When you issue an install command for a specific package, it is automatically downloaded from the internet and installed in the appropriate place on your computer.

We use the terminal or command line within Visual Studio Code to install additional Python packages. In the figure earlier in the Chapter, this is labelled as panel number 4.

To install extra Python packages, you issue install commands to a text-based window called the ‚Äúterminal‚Äù.

1.3.7.1 The Terminal in Brief

The terminal is also known as the command line and sometimes the command prompt. It was labelled 4 in the screenshot of Visual Studio Code from earlier in the chapter. The terminal is a text-based way to issue all kinds of commands to your computer (not just Python commands) and knowing a little bit about it is really useful for coding (and more) because managing packages, environments (which we haven‚Äôt yet discussed), and version control (ditto) can all be done via the terminal.

To open up the command line within Visual Studio Code, use the <kbd>‚åÉ</kbd> + <kbd>\`</kbd> keyboard shortcut (Mac) or <kbd>ctrl</kbd> + <kbd>\`</kbd> (Windows/Linux), or click "View > Terminal".

If you want to open up the command line independently of Visual Studio Code, search for "Terminal" on Mac and Linux, and "Powershell" on Windows.

Firstly, everything you can do by clicking on icons to launch programmes on your computer, you can also do via the terminal (also known as the command line). The functionality of a lot of programmes can be accessed using the command line, and some programmes only have a command line interface (CLI). This includes some that are used for data science.

The command line interacts with your operating system and can be used to create, activate, or change python installations.

Use Visual Studio Code to open a terminal window by clicking Terminal -> New Terminal on the list of commands at the very top of the window. If you have installed uv on your computer, your terminal should look something like this as your ‚Äòcommand prompt‚Äô:

your-username@your-computer current-directory %

on Mac, and the same but with ‚Äò%‚Äô replaced by ‚Äò$‚Äô on linux, and (using Powershell)

PS C:\Windows\System32>

on Windows.

You can check that uv has successfully installed Python in your current project‚Äôs folder by running

uv run python --version

You can find out more about the terminal in the chapter on {ref}wrkflow-command-line.

1.3.7.2 Installing Packages

To install any packages within the Python environment in the folder you‚Äôre using in Visual Studio Code, enter the following in Visual Studio Code‚Äôs terminal (the same place where you just ran uv run python --version).

uv add packagename

and hit return. In the above, packagename might be pandas, for example. If you have problems installing, make sure that you are connected to the internet, and that PyPI (the Python package index) isn‚Äôt blocked by your firewall or proxy.

You can see what packages you have previously installed by entering uv pip list into the command line.
