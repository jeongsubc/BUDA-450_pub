6  Classes and Methods

This section introduces classes and methods in python.

6.1 Classes

Python is a language for object-oriented programming (OOP), allowing you to structure your code using classes and objects for better organization and reusability. It has advantages over other programming approaches:

  • Provides a clear structure to programs
  • Makes code easier to maintain, reuse, and debug
  • Helps keep your code DRY (Don’t Repeat Yourself)
  • Allows you to build reusable applications with less code

In Python, a class is a blueprint for a type of objects, as it defines what an object should look like. And, an object is created based on that class as it inherits all the variables and functions defined inside that class.

Class Objects
Fruit Apple, Banana, Mango

Almost everything in Python is an object, with its properties and methods. You can create a class to store data attributes (also called the class properties) and/or methods, and use the class to handle objects of a type in a consistent and systematic way.

For example, you can define a class named MyCourse with an attribute named program, using class keyword:

# Define a class
class MyCourse:
    # data attribute
    program = 'BUDA'          # class variable shared by all instances

Now, you can create an object of the new class:

# Create an object
student_1 = MyCourse()
print("The program of student_1:", student_1.program)
The program of student_1: BUDA
# Add/modify an attribute and the value in the object 
student_1.course = "BUDA 450"    # a new attribute
print("The course code of student_1:", student_1.course)
The course code of student_1: BUDA 450

Now, you can create another object of the same class, which has the same attribute program:

# Create another object
student_2 = MyCourse()
print("The program of student_2:", student_2.program)
The program of student_2: BUDA

However, although their classes are the same, they are treated independently. So, this object student_2 doesn’t have the attribute course as it was added only to student_1. The following example returns an error message.

print("The course code of student_2:", student_2.course)

6.1.1 Method

A method is a function that “belongs to” an object. So, a method is called on an object and is specific to the object’s type. To apply a method, you can call the method’s name (and arguments), following the name of the target object to call the method on and a period (dot) . , like object_name.method_name(arguments).

# Define a class with method
class MyCourse:
    # data attribute 
    program = 'BUDA'          # class variable shared by all instances
    
    # method
    def add_course(self, course_code):
        self.course = self.program + " " + course_code

Here, self indicates the class object itself. That is, self.program call the program attribute of the object that is created through this class, locally within this class’ code block.

As a result, the function can utilize internally defined attribute that will be identically created in every MyCourse object.

For example, you can create two objects of MyCourse class, and use the same method with different arguments:

student_3 = MyCourse()
student_3.add_course("450")
print(student_3.course)
BUDA 450
student_4 = MyCourse()
student_4.add_course("455")
print(student_4.course)
BUDA 455

6.1.1.1 The __init__() Method

Almost all classes have a built-in method called __init__(), which makes the classes much useful by having an initial setup for each object. The __init__() method is always executed when the class is being initiated — when an object of the class is created.

For example, the __init__() method can be used to assign initial values to object properties, or other operations that are necessary to do when the object is being created:

# Define a class with __init__ method
class MyCourse_general:
    def __init__(self, program):
        self.program = program        # instance variable unique to each instance
    
    def add_course(self, course_code):
        self.course = self.program + " " + course_code

For example, the MyCourse_general requires an argument for program attribute to initiate (i.e., create) an object. The following objects are created with different program names, while using the same method.

# without the initial value, the object cannot be created
student_5 = MyCourse_general("BUDA")
student_5.add_course("450")
print(student_5.course)
BUDA 450
student_6 = MyCourse_general("MIST")
student_6.add_course("450")
print(student_6.course)
MIST 450
