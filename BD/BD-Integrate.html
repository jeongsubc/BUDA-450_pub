7  Data Integration

7.1 Overview

Often, data cannot be used directly for your analysis. For accurate, meaningful outcomes, data have to be prepared, prior to the application of data mining tools.

“This material outlines basic methods for preparing data in R, which is essential for analysis. It covers inspecting data, organizing data by subsetting and sorting, integrating multiple datasets, cleaning data with missing elements, and transforming data by applying mathematical operations, binning, and combining categories.”

7.2.2 Data

Let’s consider two datasets, each of which has different attributes.

df1 = pd.DataFrame(
  {
    "X1": [1, 2, 4, 4, 5],
    "X2": ["Very Satisfied", "Satisfied", "Neutral", "Dissatisfied", "Very Dissatisfied"],
    "X3": [0.5, 0.8, 0.8, 0.2, None]  # last value is missing
  }, 
  index = ["rec1", "rec2", "rec3", "rec4", "rec5"]
)

df2 = pd.DataFrame(
  {
    "X4": [50, 40, 30, 20, 10],
    "X5": [True, False, False, False, True]
  },
  index = ["rec1", "rec2", "rec3", "rec5", "rec6"]
  
)

Note that the data df1 and df2 have some records for the common objects (i.e., rec1, rec2, rec3, rec5) but different attributes.

7.3 Integrating Data

Integrating data from different sources can yield more comprehensive and reliable analysis, providing deeper insights and more reliable results.

7.3.1 Column-wise Integration

7.3.1.1 Concatenation

The DataFrame objects can be concatenated with axis=1, horizontally stacking columns:

pd.concat([df1, df2], axis=1)
X1 X2 X3 X4 X5
rec1 1.0 Very Satisfied 0.5 50.0 True
rec2 2.0 Satisfied 0.8 40.0 False
rec3 4.0 Neutral 0.8 30.0 False
rec4 4.0 Dissatisfied 0.2 NaN NaN
rec5 5.0 Very Dissatisfied NaN 20.0 False
rec6 NaN NaN NaN 10.0 True
Indices in concat

If the indices are arbitrarily set (e.g., from 0 by default), the concatenation can mislead the link between the data objects.

Let’s consider reset indices and their concatenation:

df1_reset = df1.copy().reset_index(names="rowname1")
df2_reset = df2.copy().reset_index(names="rowname2")
pd.concat([df1_reset, df2_reset], axis=1)
rowname1 X1 X2 X3 rowname2 X4 X5
0 rec1 1 Very Satisfied 0.5 rec1 50 True
1 rec2 2 Satisfied 0.8 rec2 40 False
2 rec3 4 Neutral 0.8 rec3 30 False
3 rec4 4 Dissatisfied 0.2 rec5 20 False
4 rec5 5 Very Dissatisfied NaN rec6 10 True

As you may notice, the result shows the concatenated data objects, regardless of the row names. Therefore, if records on the same row (e.g., rec4 in df1 and rec5 in df2) are for different objects, then the combined data will contain misleading information.

It’s important to check whether the indices of different DataFrame objects imply the same index system, before integrating them.

7.3.1.2 Merging (Joins)

The join1 is an operation for the systematic integration of two DataFrames of different objects (i.e., different indices), avoiding potential issues such as data integrity.

For a join of two DataFrames (say, df1 and df2 as left and right data tables), you need to specify

  • The key (e.g., a column) in each DataFrame, whose values can be used to identify each object uniquely
  • Join type
    1. Inner Join: Returns only the matching rows between two DataFrames.
    • This method is useful when you need only the common data present in both data.frames.
    1. Full Outer Join: Returns all rows from both DataFrames, filling with NaN when there are no matches.
    • This method ensures that no data is lost from either DataFrames.
    1. Left Outer Join: Returns all rows from the left DataFrame and matched rows from the right.
    • This can be useful when you want to retain all information from the primary (left) dataset.
    1. Right Outer Join: Returns all rows from the right DataFrame and matched rows from the left.
    • This can be when the secondary dataset (dat2) is more comprehensive.

You can make a join of two DataFrames, using merge() with:

  • the keys
    • on = common_column_name, when both DataFrames have the key of the same name common_name
    • left_on = left_table_column_name and right_table_column_name for the left and right data tables, respectively, when the DataFrames have the key of the different names:
    • left_index=True and right_index=True, based on the indices of the DataFrames
  • the join type
    1. how = inner for inner join
    2. how = outer for full outer join
    3. how = left for left outer join
    4. how = right for right outer join

The following show some examples of joins:

pd.merge(df1_reset, df2_reset, left_on="rowname1", right_on="rowname2", how="inner")
rowname1 X1 X2 X3 rowname2 X4 X5
0 rec1 1 Very Satisfied 0.5 rec1 50 True
1 rec2 2 Satisfied 0.8 rec2 40 False
2 rec3 4 Neutral 0.8 rec3 30 False
3 rec5 5 Very Dissatisfied NaN rec5 20 False 
pd.merge(df1, df2, left_index=True, right_index=True, how="outer")
X1 X2 X3 X4 X5
rec1 1.0 Very Satisfied 0.5 50.0 True
rec2 2.0 Satisfied 0.8 40.0 False
rec3 4.0 Neutral 0.8 30.0 False

7.3.2.1 Row Binding (Appending rows)

If both datasets have the same columns, rows can be stacked with axis=0. Order of columns doesn’t matter as long as names match (it’s handled by pandas).

This method is useful for appending similar data sets, such as adding new observations to an existing set.

pd.concat([df1, df3], axis=0)

7.3.2.2 Handling Different Columns

If two DataFrames have different columns, pd.concat() fills missing columns with NaN — like an outer join.

df3_z = df3.copy()
df3_z.columns = ['Y1','Y2','Y3']
pd.concat([df1, df3_z], axis=0)
X1 X2 X3 Y1 Y2 Y3
rec1 1.0 Very Satisfied 0.5 NaN NaN NaN
rec2 2.0 Satisfied 0.8 NaN NaN NaN
rec3 4.0 Neutral 0.8 NaN NaN NaN
rec4 4.0 Dissatisfied 0.2 NaN NaN NaN
rec5 5.0 Very Dissatisfied NaN NaN NaN NaN
rec11 NaN NaN NaN 15.0 0.01 Very Satisfied
rec12 NaN NaN NaN 25.0 0.05 Satisfied
rec13 NaN NaN NaN 35.0 0.99 Dissatisfied

To keep only the common columns (like an inner join):

common_cols = df1.columns.intersection(df3.columns)
pd.concat([df1[common_cols], df3[common_cols]], axis=0)
