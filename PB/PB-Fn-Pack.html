5  Functions and Packages

This section introduces functions and packages in Python.

5.1 Function

A function is a named block of reusable code that performs a specific task.

Python functions take inputs (called arguments) and produce outputs (called return values) by executing a defined operation. Like mathematical functions, You can pass data, known as parameters, into a function that can return data as a result.

For example, a built-in function print() outputs text to the console (or to a specified file):

print("BUDA 450")
BUDA 450

Functions help improve code clarity, reduce redundancy, and support modular design.

5.1.1 Calling a Function

To call a function, you use its name, followed by parentheses (). Inside the parentheses, you may include arguments as the inputs if needed, separated by commas , if there are multiple.

For example, the sum() function accepts arguments for a list of numbers and a value to specify a starting value for the summation:

odd_numbers = [1,3,5]
sum(odd_numbers, start=10)
19

As shown in this example, Python functions can accept two main types of arguments:

Positional arguments

  • Arguments are assigned based on their position in the function call.
  • Their order matters as the position of each argument play a different role.
# print() with two positional arguments
Whom = "World"
print("Hello", Whom)
Hello World

Keyword arguments

  • Arguments are explicitly assigned using name = value syntax, like value assignments to objects.
  • This improves clarity and allows reordering or omitting optional parameters:
# print with additional keyword argument for `sep` to specify the separator '__space__'
print("Hello", Whom, sep='__space__') 
Hello__space__World

Keyword arguments often have default values, making them optional. For instance, the print() function uses a default space ' ' for sep (see the official documentation), unless it is overwritten by the user.

Tip

Parentheses () are what we can consider for a clear distinction between functions and variables. Even when no argument is needed for a function’s operation, empty parentheses are used, referring to a function name. Even if a function takes no arguments, always use parentheses () to call it. This distinguishes functions from variables.

print()   # calling it for its operation (display none)
print     # refers to the function object itself
<function print(*args, sep=' ', end='\n', file=None, flush=False)>

5.1.1.1 Built-in Functions

Python includes a rich set of built-in functions that are always available without extra external modules. These functions handle many basic tasks, from displaying output to performing calculations or type conversions.

You can explore the full list of built-in functions in the Python’s official library documentation1.

Here are some other built-in function examples:

# Convert a number to a string
str(598)                          # Output: "598"

# Get the length of a string
len("python")                     # Output: 6

# Round a number to a specified precision
round(3.1415, 2)                  # Output: 3.14

# Find the smallest of several values
min(1, 6/8, 4/3)                  # Output: 0.75
0.75
Warning

Learning to program in Python partly means becoming familiar with its built-in functions (and many other functions in other modules related to your tasks). You don’t need to memorize them—just be aware they exist and know how to look them up when needed, like you use a dictionary for your writing.

5.1.1.2 Methods

A method is a function that belongs to an object. So, a method is called on an object and is specific to the object’s type. To apply a method, you can call the method’s name (and arguments), following the name of the target object to call the method on and a period (dot) . , like object_name.method_name(arguments).

For example, consider a string-type class2 object my_string_object and a method lower() that can be applied to the object to make all letters in the string lowercase.

my_message = "Hello World"   # a string object
print(my_message.lower())           # a lowercase version of the string object
hello world

For other methods available for strings, see the official documentation

Note

Dot notation . is also used to access attributes of an object.

For instance, if the_person is a Person object with a name attribute and a say_name() method, you would use:

  • the_person.name for the person’s name
  • the_person.say_name() to invoke the behavior method()

Think of the dot . like a possessive:

  • the_person.name means “the person’s name”;
  • the_person.say_name() means “the person’s say_name() action”.

5.1.2 Writing Functions

While Python offers many built-in and third-party functions, one of the most powerful features of the language is the ability to write your own functions. Functions allow you to encapsulate reusable logic, helping you organize code into meaningful, manageable chunks.

As Downey puts it:

“Their primary purpose is to help us organize programs into chunks that match how we think about the problem.”

Whenever you find yourself repeating the same logic, or want to improve the clarity of your program, writing a function is the right move. It reduces repetition, lowers the risk of errors, and makes your code more modular and easier to debug.

A new function can be defined with def. Here’s a simple example:

# Define a function that takes two arguments and returns a full name
def make_full_name(first_name, last_name):
    out = first_name + " " + last_name
    print(out)

# Call the function with two arguments
make_full_name("Laura", "Brown")
Laura Brown

5.1.2.1 Components in a Function

Writing a function in Python involves several key components. Each part plays a role in defining, organizing, and executing reusable code logic.

def keyword

  • Define a new function with the def keyword that signals to Python that what follows is a function definition.

Function name

  • Give your function a meaningful name that reflects its purpose.
    • One good practice is naming functions using verbs (as they do something), and naming variables using nouns (as they represent data).
  • Naming rules for general objects/variables also apply to functions (e.g., lowercase letters, underscores for separation).

Parentheses () and Parameters

  • Attach parentheses () after the function name.
  • Inside the parentheses, you may include:
    • Positional arguments: Required values, passed in order.
    • Keyword arguments: Optional values, assigned with a default.
def greet_name(name, greeting="Hello"):
    out = greeting + ', ' + name + '!'
    print(out)

print(greet_name("Emma") )                
print(greet_name("Jason", greeting="Hi")) 
Hello, Emma!
None
Hi, Jason!
None
# Functions can also take no arguments—just use empty parentheses:
def greet(name="Everyone", greeting="Hello"):
    greet_out = greeting + ', ' + name + '!'
    print(greet_out)

greet()
Hello, Everyone!

Colon :

  • End the function header line with a colon :, indicating that the body (block) of the function follows.

Function Body (Indented Block)

  • Keep indentation (typically 4 spaces) for all the body of the function.
  • It contains the operations to execute when the function is called, which can include:
    • Variable declarations
    • Other function calls
    • Control flow (if, for, etc.)

(Optional) return statement

  • Use the return keyword to specify the output of your function.
  • When return is executed, the function ends and passes the value back to where it was called.
  • The return statement can be omitted if a function does not need to return a value.
# Functions can also take no arguments—just use empty parentheses:
def greet_return(name="Everyone", greeting="Hello"):
    greet_out = greeting + ', ' + name + '!'
    print(greet_out)
    return name   # return only the name used

res = greet_return()
print("The output is", res)
Hello, Everyone!
The output is Everyone

5.1.2.2 Doc Strings

Functions are a way of abstracting behavior by organizing your code into meaningful, reusable parts. While good function and argument names help (def calc_rectangle_area(width, height) is better than def my_func(a, b, c)), naming alone isn’t always enough. To make your functions easy to understand and use (by yourself or others), it’s important to document them clearly.

A doc string (documentation string) is a special multi-line string placed right below a function definition. typically describing:

  • What the function does (at a high level), as a short abstraction (1~2 sentences);
  • What inputs (arguments) it expects;
  • What it returns (if anything).

Doc strings are enclosed in triple quotes """ and are not assigned to any variable.

def to_celsius(degrees_fahrenheit):
    """Converts Fahrenheit to Celsius and returns the result."""
    return (degrees_fahrenheit - 32) * (5/9)

If a function contains doc strings, you can access this documentation using the built-in help() function with the function’s name:

help(to_celsius)
Help on function to_celsius in module __main__:

to_celsius(degrees_fahrenheit)
    Converts Fahrenheit to Celsius and returns the result.

5.2 Modules and Packages (Libraries)

While Python includes many built-in functions, additional functionality can be organized into modules and packages

  • Module: an object that serves as an organizational unit of Python code.
  • Package: a Python module which can contain submodules or recursively, subpackages. Technically, a package is a Python module with a path attribute.

For example, Numpy is a fundamental package for many numerical operations in Python, and the package files can be found from its Github Page. In there, each .py file (e.g., numpy/numpy folder) contains functions and variables.

To use a function of a package’s module, you must first import it using the import keyword. This keeps the interpreter efficient by loading only the necessary tools into memory.

For example, math is a module that contains various useful mathematical functions. To make them available, run the import keyword with a space and module name.

# Import the built-in math module
import math

This only needs to be done once per script execution, and so is normally done at the top of the script (in a Jupyter notebook, you can include an importing code cell, or import the module at the top of the cell in which you first need it). Then, we run the code in the module:

# Call functions or access constants using dot notation
math.sqrt(25)      # 5.0
print(math.pi)     # 3.141592653589793
3.141592653589793

As you may notice, Dot notation works here just as it does for object methods. That is, math.sqrt() means “the sqrt() function in the math module.”

Package’s calling name in your code

When importing a package, its calling name can be customized (or shortened):

import math as m
m.sqrt(25)
5.0

5.2.0.1 Importing from a Custom Module

With a .py file, you can manage objects (functions and/or variables). Such custom modules can be imported in the same way.

For example, let’s consider a my_module.py file in the same directory as your script. Then, you can import the module using the filename without .py extension, as the module name:

import my_module
my_module.greet_name("Lucas")
Hello, Lucas!

5.2.1 Importing Specific Functions

You can import specific functions or variables from a module directly, rather than importing the entire module. So, in the code, you don’t need to attach the module name.

# import two objects (a function and a variable)
from math import sqrt, pi

sqrt(36)           # 6.0
print(pi)          # 3.141592653589793
3.141592653589793

Or you can import everything in the module:

from math import *
Warning

Importing all in large or unfamiliar modules is typically not recommended because the name of some function or variable may overlap with other objects. In such cases, the previous objects are overwritten, so that you lose the objects.

5.2.2 Standard Library vs. External Packages

Modules like math, random, and datetime are part of Python’s Standard Library (see all modules from the index. However, Python’s real power comes from thousands of additional libraries (or packages) developed by the community, such as pandas, numpy, and matplotlib.

To use these external libraries, you must first install them. If you’re using Anaconda, you can install packages with the conda command. For example, a package numpy can be installed by executing the following in the terminal:

conda install numpy

After installing the package to use, you can import it in the same way.

import numpy as np

data_list = [[1,2,3],[4,5,6]]  # data
np.array(data_list)         # a numpy ndarray object
array([[1, 2, 3],
       [4, 5, 6]])
