Appendix B — Proximity

This section introduces the concept of proximity—how similar or dissimilar data objects are—and explains how to measure it in practice.

B.1 What is Proximity?

Proximity measures, including similarity and dissimilarity, play a central role in data analysis and form the foundation of many data mining and machine learning algorithms.

  • Proximity, a general term referring to either similarity or dissimilarity between two data objects.

  • Similarity indicates how alike two data objects are.

    • Expressed as a numerical value, typically between 0 and 1.
    • Higher values indicate greater similarity.

  • Dissimilarity indicates how different two data objects are.

    • Expressed as a numerical value, often ranging from 0 to ∞.
    • Lower values indicate greater similarity.

B.1.1 Choosing a Measure

The choice of proximity measure depends on the type of attributes:

  • For nominal attributes, measures such as the Simple Matching Coefficient (SMC) and the Jaccard Index are commonly used.

  • For numerical attributes, distance-based measures are most common, such as Euclidean distance, Manhattan distance, and Minkowski distance

B.2 Measures for Single-Attribute Data

When two data objects have only a single attribute, their proximity can be assessed by directly comparing their attribute values. The approach depends on the type of attribute: nominal, ordinal, or numerical.

B.2.1 Nominal Attributes

For nominal (categorical) attributes, we can compare values directly using comparison operators1.

  • The equality operator == checks if two values are the same.
  • The result is a Boolean (True or False).
  • By converting the Boolean to an integer, we obtain a numerical similarity (1 if equal, 0 otherwise).
# Define nominal attribute values
x = 'WV'
y = 'PA'
z = 'WV'

# Similarity (1 if equal, 0 otherwise)
sim_xy = int(x == y)
sim_xz = int(x == z)

print("Similarity between x and y:", sim_xy)
print("Similarity between x and z:", sim_xz)
Similarity between x and y: 0
Similarity between x and z: 1

Dissimilarity can be obtained using the negated equal operator !=, again converted to integers:

# Dissimilarity (1 if different, 0 otherwise)
dissim_xy = int(x != y)
dissim_xz = int(x != z)

print("Dissimilarity between x and y:", dissim_xy)
print("Dissimilarity between x and z:", dissim_xz)
Dissimilarity between x and y: 1
Dissimilarity between x and z: 0

B.2.2 Ordinal Attributes

For ordinal attributes, proximity is based on the rank order of values rather than the raw categories themselves.

To compute proximity:

  1. Map ordinal values to numeric ranks (e.g., integer labels based on order).
  2. Compare ranks using distance or difference-based measures.

As an example, consider grades for five students. Suppose we rank the grades alphabetically2. We can obtain these ranks in Python with the rankdata() function from scipy.stats:

from scipy.stats import rankdata

# Sample ordinal data
dat_grade = ["A", "B", "C", "D", "C"]

# Rank alphabetically
dat_grade_rank = rankdata(
    dat_grade, 
    method='min'     # method='min' assigns ties the lowest rank
)
print("Ranked grades:", dat_grade_rank)
Ranked grades: [1 2 3 5 3]

Once the ranks are assigned, we can measure proximity between two objects. For instance, the dissimilarity between the first and third elements is the normalized rank difference:

# Extract ranks of the first and third items
x = dat_grade_rank[0]
y = dat_grade_rank[2]
n = len(dat_grade)

# Dissimilarity: normalized absolute difference
dis_xy = abs(x - y) / (n - 1)
print("Dissimilarity between x and y:", dis_xy)
Dissimilarity between x and y: 0.5

Here, abs(x - y) gives the absolute rank difference. Dividing by (n - 1) normalizes the dissimilarity to fall between 0 and 1.

Then, similarity can be derived from dissimilarity by applying a mathematical transformation, as the complement.

# Similarity: 1 - dissimilarity
sim_xy = 1 - dis_xy
print("Similarity between x and y:", sim_xy)
Similarity between x and y: 0.5

B.2.3 Numerical Attributes

For numerical attributes, proximity can be computed directly using numeric operations.

A common dissimilarity measure is the absolute difference between two values:

# Sample numerical values
x, y = 65, 45

# Dissimilarity: absolute difference
dis_xy = abs(x - y)
print("Dissimilarity:", dis_xy)
Dissimilarity: 20
Note

Similarity is often derived from dissimilarity by applying a mathematical transformation. Different transformations reflect different assumptions about how similarity should decrease as difference grows.

  1. Negated difference as a simple but unbounded transformation with \(f(x)= -x\)::
# Similarity as negative dissimilarity
sim_xy = -dis_xy
print("Similarity (negated difference):", sim_xy)
Similarity (negated difference): -20
  1. Inverse transformation that maps dissimilarity into a bounded similarity score in the range \((0, 1]\):
# Similarity using inverse transformation
sim_xy = 1 / (1 + dis_xy)
print("Similarity (inverse):", sim_xy)
Similarity (inverse): 0.047619047619047616
  1. Exponential decay that strongly penalizes larger differences, with \(f(x)=e^{-x}\):
import math
# Similarity using exponential decay
sim_xy = math.exp(-dis_xy)
print("Similarity (exponential decay):", sim_xy)
Similarity (exponential decay): 2.061153622438558e-09

B.3 Measures for Multiple-Attribute Data

In most real-world applications, each data object is described by multiple attributes (features).
Measuring proximity—whether similarity or dissimilarity—between these multi-attribute objects is essential for tasks such as clustering, classification, and nearest-neighbor search.

When multiple attributes are involved, we consider the entire feature vector of each object and apply a mathematical function that aggregates the differences across all attributes.

B.3.1 Simple Matching Coefficient (SMC)

The Simple Matching Coefficient (SMC) is commonly used for binary or nominal attributes. It calculates the proportion of attributes where two objects match.

B.3.1.1 SMC for a Pair of Objects

Suppose we have two data records xi and xj, each with five binary attributes:

# Define two categorical vectors
xi = ['Yes', 'No', 'No', 'Yes', 'No']
xj = ['Yes', 'Yes', 'No', 'Yes', 'No']

We compare the attributes element-wise with the equality operator == that returns a Boolean value (True if equal, False otherwise):

# Element-wise comparison (Boolean)
matches = [a == b for a, b in zip(xi, xj)]
print("Matching positions:", matches)
Matching positions: [True, False, True, True, True]

Since Python treats True and False as numeric 1 and 0, respectively, we can count matches by summing the Boolean values in the list. Then, dividing by the total number of attributes gives the SMC:

# Compute simple matching coefficient
numer = sum(matches)       # Number of matching attributes
denom = len(xi)            # Total number of attributes
smc = numer / denom        # Simple Matching Coefficient

print("Simple Matching Coefficient:", smc)
Simple Matching Coefficient: 0.8

For convenience, the computation can be written as a one-liner:

# One-liner SMC computation
smc = sum([a == b for a, b in zip(xi, xj)]) / len(xi)
print("SMC (one-liner):", smc)
SMC (one-liner): 0.8
Note

The Simple Matching Coefficient is appropriate when:

  • All attributes are equally important, and
  • Presence and absence are treated symmetrically (no special emphasis on one category).

B.3.1.2 SMC for Multiple Objects

Now, let’s extend the idea of SMC to a dataset with more than two objects.

Suppose we have three records xi, xj, and xk, each described by five binary attributes:

import pandas as pd

# Define three categorical records
xi = ['Yes', 'No', 'No', 'Yes', 'No']
xj = ['Yes', 'Yes', 'No', 'Yes', 'No']
xk = ['No', 'Yes', 'Yes', 'Yes', 'No']

# Combine into a DataFrame
dat1 = pd.DataFrame(
    [xi, xj, xk], 
    index=['xi', 'xj', 'xk'],
    columns=['X1', 'X2', 'X3', 'X4', 'X5']
)
dat1
X1 X2 X3 X4 X5
xi Yes No No Yes No
xj Yes Yes No Yes No
xk No Yes Yes Yes No
Convert to Binary

To compute similarity measures efficiently, it helps to convert categorical attributes into binary form.

For example, we can encode whether each attribute equals Yes (presence) or No (absence):

# Convert to binary (logical) format based on 'Yes' responses
dat1_bin = dat1 == "Yes"
dat1_bin
X1 X2 X3 X4 X5
xi True False False True False
xj True True False True False
xk False True True True False
Compute Pairwise SMCs

Python does not provide a built-in "simple matching" metric in typical data science packages.

However, we can leverage the Hamming distance as an equivalent measure for binary attributes, since SMC = 1 – Hamming distance:

  • Hamming distance: proportion of attributes where two objects differ.
  • Simple Matching Coefficient (SMC): proportion of attributes where two objects match.

The function pdist() in the module scipy.spatial.distance in scipy package computes pairwise distances between data objects, and then convert them to similarities.

from scipy.spatial.distance import pdist, squareform

# Pairwise dissimilarity (Hamming distance)
hamm_condensed = pdist(dat1_bin, metric='hamming')
hamm_matrix = squareform(hamm_condensed)
print(hamm_matrix)
[[0.  0.2 0.6]
 [0.2 0.  0.4]
 [0.6 0.4 0. ]]

The matrix hamm_matrix above shows the dissimilarities (fraction of mismatches) between each pair of objects. To obtain the Simple Matching Coefficient, subtract from 1:

# Pairwise similarity (SMC = 1 - Hamming)
smc_matrix = 1 - hamm_matrix
print(smc_matrix)
[[1.  0.8 0.4]
 [0.8 1.  0.6]
 [0.4 0.6 1. ]]
Accessing Specific Similarities

You can extract a particular similarity value directly from the matrix. For example, the similarity between xi (row 0) and xk (row 2):

i, j = 0, 2
smc_matrix[i, j]
np.float64(0.4)
Custom Function

Sometimes, the similarity or distance measure you need may not be directly available in existing Python libraries. In such cases, writing a custom function is often the most flexible solution.

Below, you can create a custom function simple_matching() that calculates the SMC between two binary records x and y:

# Define a custom function for SMC
def simple_matching(x, y):
    return sum(x == y) / len(x)

Using the function, the SMC matrix can be obtained:

# Compute pairwise similarity matrix manually
smc_matrix_custom = pd.DataFrame(index=dat1_bin.index, columns=dat1_bin.index, dtype=float)
for i in dat1_bin.index:
    for j in dat1_bin.index:
        smc_matrix_custom.loc[i, j] = simple_matching(dat1_bin.loc[i], dat1_bin.loc[j])

smc_matrix_custom
xi xj xk
xi 1.0 0.8 0.4
xj 0.8 1.0 0.6
xk 0.4 0.6 1.0

This approach directly computes a full similarity matrix across all objects, without relying on built-in metrics.

TipFlexibility of Approaches

A given Python task can often be accomplished in several ways. Choosing the right approach depends on factors such as:

  • Performance (computational speed for large datasets)
  • Ease of coding (simplicity and readability)
  • Compactness (shorter vs. more explicit code)
  • Compatibility (with other tools, packages, or programming languages)

In such situations,

  • You may use existing package functions, following the format required (so that you may need to do some preprocessing).
  • You may define a custom function, giving you full control of the calculation (and possibly incorporating data preprocessing).
  • Or, you may combine both strategies, as shown below.
# A custom function
def simple_nonmatching(x, y):
    return sum(x != y) / len(x)

# Compute pairwise similarity matrix with pdist in scipy
smc_condensed = pdist(dat1_bin, metric=simple_nonmatching)
smc_matrix_mix = 1 - squareform(smc_condensed)
print(smc_matrix)
[[1.  0.8 0.4]
 [0.8 1.  0.6]
 [0.4 0.6 1. ]]

To make such a decision, always consider:

  • Do I need to preprocess the data to fit the requirements of a library function?
  • Would a custom function be simpler for my dataset and teaching purpose?
  • Can I combine both approaches to balance flexibility and efficiency?

B.3.2 Jaccard Coefficient

The Jaccard coefficient (also called the Jaccard index) measures the similarity between two records by focusing only on shared positive values.

Unlike the SMC, the Jaccard coefficient ignores negative–negative matches (False–False) since the absence of attributes is not considered evidence of similarity in many applications (e.g., market basket analysis).

Let’s compare the two binary records xj and xk from our earlier dat1_bin dataset:

# Extract binary rows for xj and xk
xj_bin = dat1_bin.loc['xj']
xk_bin = dat1_bin.loc['xk']

# Display both for comparison
print(
    pd.DataFrame([xj_bin, xk_bin], index=['xj', 'xk'])
)
       X1    X2     X3    X4     X5
xj   True  True  False  True  False
xk  False  True   True  True  False

B.3.2.1 Counting Possible Combinations

For each attribute, the pair (xj, xk) can take one of four combinations:

  • True–True → positive-positive (count_pp)
  • True–False → positive-negative (count_pn)
  • False–True → negative-positive (count_np)
  • False–False → negative-negative (count_nn)

We can count these cases as follows:

# Count different value combinations
count_pp = ((xj_bin == True) & (xk_bin == True)).sum()
count_pn = ((xj_bin == True) & (xk_bin == False)).sum()
count_np = ((xj_bin == False) & (xk_bin == True)).sum()
count_nn = ((xj_bin == False) & (xk_bin == False)).sum()

# Show as a 2x2 matrix in a DataFrame
print(
    pd.DataFrame(
        np.array([[count_pp, count_pn], [count_np, count_nn]]),
        index=["True", "False"],
        columns=["True", "False"]
    )
)
       True  False
True      2      1
False     1      1

Alternatively, you can get this same result using crosstab() in pandas:

# Cross-tabulation using pandas
print(
    pd.crosstab(xj_bin.astype(bool), xk_bin.astype(bool), rownames=['xj'], colnames=['xk'])
)
xk     False  True 
xj                 
False      1      1
True       1      2
Note

We use .astype(bool) to ensure the values are treated as Boolean categories (True/False), which makes the crosstab() result clearer.

B.3.2.2 Compute Jaccard Similarity

The Jaccard coefficient measures similarity by focusing only on attributes where at least one record is positive (True).
Formally:

\[ J(x_j, x_k) = \frac{\#( \text{Positive–Positive} )}{\#( \text{Positive–Positive, Positive–Negative, Negative–Positive} )} \]

That is, the similarity is the ratio of shared positives to all attributes with at least one positive.

numer = count_pp
denom = count_pp + count_pn + count_np
jaccard_coeff = numer / denom
jaccard_coeff
np.float64(0.5)

Or, you can also calculate the Jaccard coefficient matrix using jaccard_score() in sklearn.metrics:

from sklearn.metrics import jaccard_score

# Apply Jaccard similarity on binary rows (must convert to numpy or pandas arrays internally)
jaccard_score(dat1_bin.loc['xj'], dat1_bin.loc['xk'], average='binary')
np.float64(0.5)

Alternatively, you can compute the whole Jaccard distance matrix using scipy’s pdist():

# Compute Jaccard distance (1 - similarity)
jaccard_dist = pdist(dat1_bin, metric='jaccard')
jaccard_sim_matrix = 1 - squareform(jaccard_dist)

# Format as a DataFrame
print(pd.DataFrame(jaccard_sim_matrix, index=dat1_bin.index, columns=dat1_bin.index))
          xi        xj    xk
xi  1.000000  0.666667  0.25
xj  0.666667  1.000000  0.50
xk  0.250000  0.500000  1.00
Warning

In scipy, pdist(..., metric='jaccard') returns the distance (i.e., 1 - similarity).

B.3.3 Numerical Attributes

Consider a dataset with three objects—xi, xj, and xk—each described by two numerical attributes:

  • X1: Average time spent online (hours)
  • X2: Number of orders during the last month
import pandas as pd

# Create a DataFrame with numerical attributes
dat2 = pd.DataFrame({
    'X1': [2, 3, 3],  # Average time spent online
    'X2': [3, 0, 4]   # Number of orders during the last month
}, index=['xi', 'xj', 'xk'])
print(dat2)
    X1  X2
xi   2   3
xj   3   0
xk   3   4

For demonstration, let’s extract two records (xi and xj) as vectors:

# Extract specific data points as Series
xi = dat2.loc['xi']
xj = dat2.loc['xj']

print("xi:", xi.to_list())
print("xj:", xj.to_list())
xi: [2, 3]
xj: [3, 0]

With numerical attributes, we can compute a variety of distance and similarity measures such as

  • Euclidean distance → straight-line (L2) distance between vectors
  • Manhattan distance → sum of absolute differences (L1 distance)
  • Cosine similarity → similarity based on the angle between vectors
  • Correlation coefficient → similarity based on linear relationship

B.3.3.1 Distance

Distances are commonly used as dissimilarity measures between numerical data objects.

Euclidean Distance

The Euclidean distance between two objects \(\mathbf{x}_i\) and \(\mathbf{x}_j\) is defined as:

\[ d(\mathbf{x}_i, \mathbf{x}_j) = \sqrt{ \sum_{k=1}^{N}(x_{ik} - x_{jk})^2 }, \]

Instead of manual computation, we can use built-in functions from scipy.spatial.distance or sklearn.metrics

You can use functions from scipy.spatial.distance or sklearn.metrics3 to compute the distance between rows:

from scipy.spatial.distance import euclidean

# Compute Euclidean distance between xi and xj
euclidean(dat2.loc['xi'], dat2.loc['xj'])
np.float64(3.1622776601683795)
# Combine xi and xj into a matrix
dat2_ij = dat2.loc[['xi', 'xj']]

# Compute Euclidean distance
euclidean(dat2_ij.loc['xi'], dat2_ij.loc['xj'])
np.float64(3.1622776601683795)

To compute pairwise Euclidean distances across all rows, use scipy.spatial.distance.pdist() with metric='euclidean':

from scipy.spatial.distance import pdist, squareform

# Pairwise Euclidean distances
dist_matrix_euclidean = pd.DataFrame(
    squareform(pdist(dat2, metric='euclidean')),
    index=dat2.index, columns=dat2.index
)
print(dist_matrix_euclidean)
          xi        xj        xk
xi  0.000000  3.162278  1.414214
xj  3.162278  0.000000  4.000000
xk  1.414214  4.000000  0.000000
NoteSteps for manual computation and the codes
  1. Compute element-wise differences
  2. Square the differences
  3. Sum the squared differences
  4. Take the square root of the sum
diff = xi - xj                              # 1) xik - xjk for each attribute k
diff_sqr = diff ** 2                        # 2) (xik - xjk)^2
diff_sqr_sum = diff_sqr.sum()               # 3) sum((xi - xj)^2)
euclidean_manual = np.sqrt(diff_sqr_sum)    # 4) sqrt(sum((xi - xj)^2))
euclidean_manual
np.float64(3.1622776601683795)

B.3.3.2 Other Distance Metrics

Besides Euclidean, several other distance metrics are useful in practice:

Manhattan Distance (City Block), \(d(\mathbf{x}_i, \mathbf{x}_j) = \sum_{k=1}^{N} |x_{ik} - x_{jk}|\):

# Manhattan (L1) distance matrix
dist_matrix_manhattan = pd.DataFrame(
    squareform(pdist(dat2, metric='cityblock')),
    index=dat2.index, columns=dat2.index
)
print(dist_matrix_manhattan)
     xi   xj   xk
xi  0.0  4.0  2.0
xj  4.0  0.0  4.0
xk  2.0  4.0  0.0

Minkowski Distance (Generalized form with parameter p), \(d(\mathbf{x}_i, \mathbf{x}_j) = \Bigg( \sum_{k=1}^{N} |x_{ik} - x_{jk}|^p \Bigg)^{\tfrac{1}{p}}\):

# Minkowski distance with p = 1.5
dist_matrix_minkowski = pd.DataFrame(
    squareform(pdist(dat2, metric='minkowski', p=1.5)),
    index=dat2.index, columns=dat2.index
)
print(dist_matrix_minkowski)
          xi        xj        xk
xi  0.000000  3.373505  1.587401
xj  3.373505  0.000000  4.000000
xk  1.587401  4.000000  0.000000

Chebyshev Distance (Minkowski with \(p\rightarrow\infty\), \(d(\mathbf{x}_i, \mathbf{x}_j) = \max_{k} \, |x_{ik} - x_{jk}|\):

# Chebyshev (maximum coordinate difference)
dist_matrix_chebyshev = pd.DataFrame(
    squareform(pdist(dat2, metric='chebyshev')),
    index=dat2.index, columns=dat2.index
)
print(dist_matrix_chebyshev)
     xi   xj   xk
xi  0.0  3.0  1.0
xj  3.0  0.0  4.0
xk  1.0  4.0  0.0
Note

All of these distance metrics are available through scipy.spatial.distance.pdist(), which provides an efficient way to compute pairwise distances for entire datasets.
