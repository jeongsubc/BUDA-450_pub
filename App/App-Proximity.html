Appendix A — Proximity

A.1 Overview

Proximity measures—including similarity and dissimilarity (e.g., distance—are essential not only for data analysis but also as core components in many data mining algorithms. This session introduces methods for measuring proximity between data objects with single or multiple attributes, tailored to the type of data. For nominal attributes, techniques such as the Simple Matching Coefficient and Jaccard Index are applied, while numerical attributes rely on distance measures like Euclidean, Manhattan, and Minkowski distances. To ensure meaningful comparisons across attributes with different scales, normalization techniques such as standardization and min-max rescaling are also discussed.

A.3 Measures for Single Attribute Data

When two data objects have only a single attribute, their proximity can be measured in a function of a direct comparison between the attribute values.

When two data objects have only a single attribute, their proximity can be assessed by directly comparing their attribute values.

A.3.1 Nominal Attributes

For nominal (categorical) attributes, logical operators can be used to compare values. In Python, the equality operator == checks whether the values are the same, and the result is a boolean (True or False). To quantify similarity, we can cast these boolean results to integers (1 for True, 0 for False):

# Define nominal attribute values
x = 'WV'
y = 'PA'
z = 'WV'

# Similarity (1 if equal, 0 otherwise)
sim_xy = int(x == y)
sim_xz = int(x == z)

print("Similarity between x and y:", sim_xy)
print("Similarity between x and z:", sim_xz)
Similarity between x and y: 0
Similarity between x and z: 1

To compute dissimilarity, we use the inequality operator != and apply the same numeric conversion:

# Dissimilarity (1 if different, 0 otherwise)
dissim_xy = int(x != y)
dissim_xz = int(x != z)

print("Dissimilarity between x and y:", dissim_xy)
print("Dissimilarity between x and z:", dissim_xz)
Dissimilarity between x and y: 1
Dissimilarity between x and z: 0

A.3.2 Ordinal Attributes

Proximity measures for ordinal attributes are based on the rankings of attribute values. That is, ordinal values are mapped to numeric labels (e.g., the integer coding) according to their rank, and then the proximity is assessed. For example, let dat_grade represent the grades of five students in an ordinal attribute. If we rank the values alphabetically1, we can use Python’s rankdata() function from the scipy.stats module.

from scipy.stats import rankdata

# Sample ordinal data
dat_grade = ["A", "B", "C", "D", "C"]

# Rank the data alphabetically; method='min' handles ties by assigning the lowest rank
dat_grade_rank = rankdata(dat_grade, method='min')
print("Ranked grades:", dat_grade_rank)
Ranked grades: [1 2 3 5 3]

With these ranks, the (dis)similarity between two data objects can be computed. For instance, the dissimilarity between the first and third elements is calculated as the normalized rank difference:

# Extract ranks of the first and third items
x = dat_grade_rank[0]
y = dat_grade_rank[2]
n = len(dat_grade)

# Dissimilarity: normalized absolute difference
dis_xy = abs(x - y) / (n - 1)
print("Dissimilarity between x and y:", dis_xy)

# Similarity: 1 - dissimilarity
sim_xy = 1 - dis_xy
print("Similarity between x and y:", sim_xy)
Dissimilarity between x and y: 0.5
Similarity between x and y: 0.5

Here, abs() returns the absolute value of the difference, and the result is scaled by dividing by n-1 to normalize the range between 0 and 1.

A.3.3 Numerical attributes

For numerical attributes, proximity can be directly computed using numerical operations.

A common measure of dissimilarity is the absolute difference between two values.

# Sample numerical values
x = 65
y = 45

# Dissimilarity: absolute difference
dis_xy = abs(x - y)
print("Dissimilarity:", dis_xy)
Dissimilarity: 20

A corresponding similarity can then be defined as the negative of that dissimilarity or transformed via other mathematical functions.

# Similarity as negative dissimilarity
sim_xy = -dis_xy
print("Similarity (negated difference):", sim_xy)
Similarity (negated difference): -20

Other mathematical transformations can be used to define similarity on a bounded or decaying scale:

# Similarity using inverse transformation
sim_xy = 1 / (1 + dis_xy)
print("Similarity (inverse):", sim_xy)
Similarity (inverse): 0.047619047619047616

Here, math.exp() returns the exponential of the input (i.e., \(e^x\)) where \(e\) is the base of the natural logarithm. This form of similarity decays rapidly as the dissimilarity increases.

import math
# Similarity using exponential decay
sim_xy = math.exp(-dis_xy)
print("Similarity (exponential decay):", sim_xy)
Similarity (exponential decay): 2.061153622438558e-09

A.4 Measures for Multiple Attribute Data

In most real-world data mining applications, each data object is described by multiple attributes (features). Measuring proximity—such as similarity or dissimilarity—between these multi-attribute objects is essential for tasks like clustering, classification, and nearest-neighbor search.

When multiple attributes are involved, we consider the entire feature vector of each object and apply a mathematical function that aggregates the differences across all attributes.

For example, common dissimilarity measures include:

  • Euclidean distance (straight-line distance)
  • Manhattan distance (sum of absolute differences)
  • Minkowski distance (generalized form)
  • Cosine dissimilarity (based on angle between vectors)

We’ll explore these with Python examples in the following sections.

A.4.1 Nominal Attributes — Simple Matching Coefficient

Let’s consider two data records, xi and xj, each with five binary (two-class nominal) attributes:

# Define two categorical vectors
xi = ['Yes', 'No', 'No', 'Yes', 'No']
xj = ['Yes', 'Yes', 'No', 'Yes', 'No']

In Python, we can use list comprehension or NumPy to compare elements element-wise. The equality operator == applied to each pair of attributes produces a list of Boolean values:

# Element-wise comparison (Boolean)
matches = [a == b for a, b in zip(xi, xj)]
print("Matching positions:", matches)
Matching positions: [True, False, True, True, True]

When performing numerical operations, Python treats True as 1 and False as 0. Thus, summing these values counts the number of matches, and dividing by the total number of attributes gives the Simple Matching Coefficient (SMC):

# Compute simple matching coefficient
numer = sum(matches)       # Number of matching attributes
denom = len(xi)            # Total number of attributes
smc = numer / denom        # Simple Matching Coefficient

print("Simple Matching Coefficient:", smc)
Simple Matching Coefficient: 0.8

Of course, this can also be done concisely in a single line:

# One-liner SMC computation
smc = sum([a == b for a, b in zip(xi, xj)]) / len(xi)
print("SMC (one-liner):", smc)
SMC (one-liner): 0.8

The Simple Matching Coefficient is useful when all attributes are equally important and symmetrically valued (i.e., no distinction between presence and absence).

A.4.1.1 Simple Matching Coefficient for Multiple Objects

Now, let’s consider a dataset that consists of three objects xi, xj, and xk:

import pandas as pd

# Define three categorical records
xi = ['Yes', 'No', 'No', 'Yes', 'No']
xj = ['Yes', 'Yes', 'No', 'Yes', 'No']
xk = ['No', 'Yes', 'Yes', 'Yes', 'No']

# Combine into a DataFrame
dat1 = pd.DataFrame([xi, xj, xk], index=['xi', 'xj', 'xk'],
                    columns=['X1', 'X2', 'X3', 'X4', 'X5'])

When calculating similarity measures for multiple records, it is helpful to define a function for repeated use. While Python doesn’t have a built-in “simple matching” function like R’s proxy::simil(), we can implement it or use existing tools after converting the data.

Convert Categorical to Binary

We need to convert each categorical column into binary form, for example by checking whether each value equals Yes:

# Convert to binary (logical) format based on 'Yes' responses
dat1_bin = pd.DataFrame({
    'X1': dat1['X1'] == 'Yes',
    'X2': dat1['X2'] == 'Yes',
    'X3': dat1['X3'] == 'Yes',
    'X4': dat1['X4'] == 'Yes',
    'X5': dat1['X5'] == 'Yes'
}, index=dat1.index)
dat1_bin
X1 X2 X3 X4 X5
xi True False False True False
xj True True False True False
xk False True True True False
Compute Simple Matching Coefficients

https://docs.scipy.org/doc/scipy/reference/generated/scipy.spatial.distance.pdist.html#scipy.spatial.distance.pdist

Python does not have a built-in “simple matching” metric in sklearn or scipy, but you can define and extend your own as shown above. Alternatively, similarity/distance measures like Jaccard, Hamming, or cosine similarity can be used for different purposes depending on your data.

from scipy.spatial.distance import pdist, squareform

# Compute the pairwise distance (i.e., dissimilarity) matrix 
hamm_condensed = pdist(dat1_bin, metric='hamming')
hamm_matrix = squareform(hamm_condensed)
hamm_matrix

# Compute the pairwise coefficient (i.e., similarity) matrix 
smc_matrix = 1 - hamm_matrix
smc_matrix
array([[1. , 0.8, 0.4],
       [0.8, 1. , 0.6],
       [0.4, 0.6, 1. ]])

To access specific similarity values, for example between xi and xj, you can do:

i, j = 0, 2
smc_matrix[i, j]
np.float64(0.4)
Custom function

Sometimes, the measure you want to use may be unavailable in such existing function/package. Then, creating a custom function can be a good solution.

For example, you can create a custom function simple_matching() that takes two input arguments x and y for the calculation for the matching coefficient between two objects x and y.

We can now compute the simple matching coefficient using the formula:

# Define a custom function for SMC
def simple_matching(x, y):
    return sum(x == y) / len(x)

# Compute pairwise similarity matrix manually
smc_matrix = pd.DataFrame(index=dat1_bin.index, columns=dat1_bin.index, dtype=float)
for i in dat1_bin.index:
    for j in dat1_bin.index:
        smc_matrix.loc[i, j] = simple_matching(dat1_bin.loc[i], dat1_bin.loc[j])

smc_matrix
xi xj xk
xi 1.0 0.8 0.4
xj 0.8 1.0 0.6
xk 0.4 0.6 1.0
Tip

A task with Python can be performed in many possible approaches, and an approach is chosen based on a different criterion (e.g., computing, coding difficulty, compactness, compatibility with other code/software/language, etc.). For that, you may select the way best for you:

  • based on the existing function(s) in some package, following the format required (so that you may need to do some preprocessing).
  • based on a custom function, in which you can specify all needed for the calculation (in this example, you could get the result directly by using the original categorical data without the preprocessing).
  • or, based on some mix like an example below:
# A custom function
def simple_nonmatching(x, y):
    return sum(x != y) / len(x)

# Compute pairwise similarity matrix with pdist in scipy
smc_condensed = pdist(dat1_bin, metric=simple_nonmatching)
smc_matrix = 1 - squareform(smc_condensed)
smc_matrix
array([[1. , 0.8, 0.4],
       [0.8, 1. , 0.6],
       [0.4, 0.6, 1. ]])

A.4.1.2 Jaccard Coefficient

The Jaccard coefficient (or Jaccard index) measures the similarity between two records by focusing on the co-occurring values among the values that exist in any of the records. In other words, the similarity between two binary vectors focuses on the shared positive values.

Let’s compare the two binary records xj and xk from our earlier dat1_bin dataset:

# Extract binary vectors for xj and xk
xj_bin = dat1_bin.loc['xj']
xk_bin = dat1_bin.loc['xk']

# Display both for comparison
pd.DataFrame([xj_bin, xk_bin], index=['xj', 'xk'])
X1 X2 X3 X4 X5
xj True True False True False
xk False True True True False

There are four possible value combinations per attribute across the two records:

  • True–True → positive-positive (count_pp)
  • True–False → positive-negative (count_pn)
  • False–True → negative-positive (count_np)
  • False–False → negative-negative (count_nn)

We can count each case manually:

# Count different value combinations
count_pp = ((xj_bin == True) & (xk_bin == True)).sum()
count_pn = ((xj_bin == True) & (xk_bin == False)).sum()
count_np = ((xj_bin == False) & (xk_bin == True)).sum()
count_nn = ((xj_bin == False) & (xk_bin == False)).sum()

# Show as a 2x2 matrix
import numpy as np
np.array([[count_pp, count_pn], [count_np, count_nn]])
array([[2, 1],
       [1, 1]])

Alternatively, you can get this same result using a crosstab (like R’s table()):

# Crosstabulation using pandas
pd.crosstab(xj_bin.astype(bool), xk_bin.astype(bool),
            rownames=['xj'], colnames=['xk'])
xk False True
xj
False 1 1
True 1 2
Note

To ensure proper handling of boolean categories, we explicitly cast values using .astype(bool).

Then, Jaccard similarity is computed as:

\[ J(x_j, x_k) = \frac{\text{# of } True\text{-}True}{\text{# of attributes with at least one } True} \]

That is:

numer = count_pp
denom = count_pp + count_pn + count_np
jaccard_coeff = numer / denom
jaccard_coeff
np.float64(0.5)

You can also calculate the Jaccard coefficient matrix using sklearn.metrics:

from sklearn.metrics import jaccard_score

# Apply Jaccard similarity on binary rows (note: must convert to numpy)
jaccard_score(dat1_bin.loc['xj'], dat1_bin.loc['xk'], average='binary')
np.float64(0.5)

Or compute pairwise Jaccard distance between all rows using scipy’s pdist sklearn’s pairwise_distances:

dat1_bin
# Compute Jaccard distance (1 - similarity)
jaccard_dist = pdist(dat1_bin, metric='jaccard')
jaccard_sim_matrix = 1 - squareform(jaccard_dist)

# Format as a DataFrame
pd.DataFrame(jaccard_sim_matrix, index=dat1_bin.index, columns=dat1_bin.index)
xi xj xk
xi 1.000000 0.666667 0.25
xj 0.666667 1.000000 0.50
xk 0.250000 0.500000 1.00
Warning

In scikit-learn, pairwise_distances(…, metric=‘jaccard’) returns the distance (i.e., 1 - similarity).

A.4.2 Numerical Attributes

A.4.2.1 Distance

Distances are commonly used as dissimilarity measures between numerical data objects.

Using the formula
\[ d(\mathbf{x}_i, \mathbf{x}_j) = \sqrt{ \sum_{k=1}^{N}(x_{ik} - x_{jk})^2 }, \]

we can manually compute the Euclidean distance between xi and xj in the order:

  1. Compute element-wise difference
  2. Square of differences
  3. Sum of squared differences
  4. Square root of the sum
diff = xi - xj                              # 1) xik-xjk for each attribute k
diff_sqr = diff ** 2                        # 2) (xik-xjk)^2 for each k
diff_sqr_sum = diff_sqr.sum()               # 3) sum((xi-xj)^2)
euclidean_manual = np.sqrt(diff_sqr_sum)    # 4) sqrt(sum((xi-xj)^2))
euclidean_manual
np.float64(3.1622776601683795)
Euclidean Distance Using Built-in Function

You can also use scipy.spatial.distance or sklearn.metrics2 to compute the distance between rows:

from scipy.spatial.distance import euclidean

# Combine xi and xj into a matrix
dat2_ij = dat2.loc[['xi', 'xj']]

# Compute Euclidean distance
euclidean(dat2_ij.loc['xi'], dat2_ij.loc['xj'])
np.float64(3.1622776601683795)

To compute pairwise distances for all rows, use scipy.spatial.distance.pdist with squareform:

from scipy.spatial.distance import pdist, squareform

# Euclidean distances for all pairs
dist_matrix_euclidean = pd.DataFrame(
    squareform(pdist(dat2, metric='euclidean')),
    index=dat2.index, columns=dat2.index
)
dist_matrix_euclidean
